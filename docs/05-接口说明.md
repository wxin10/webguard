# 05-接口说明

## 通用响应格式

系统接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

`code=0` 表示成功，非 0 表示业务失败。HTTP 状态码用于表达请求层面的成功、鉴权失败、参数错误或服务异常。

## 检测接口

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/scan/url` | 根据 URL 发起风险检测 |
| POST | `/api/v1/scan/page` | 根据网页特征发起风险检测 |
| POST | `/api/v1/plugin/analyze-current` | 插件提交当前页面特征进行检测 |

URL 检测请求示例：

```json
{
  "url": "https://example.com"
}
```

页面检测请求示例：

```json
{
  "url": "https://example.com/login",
  "title": "Example Login",
  "visible_text": "login password account",
  "button_texts": ["Login"],
  "input_labels": ["Password"],
  "form_action_domains": ["example.com"],
  "has_password_input": true,
  "source": "web"
}
```
