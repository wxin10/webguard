# 04-技术实现说明

## 后端实现

后端使用 FastAPI 实现 REST API，所有接口通过统一响应结构返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

后端通过 SQLAlchemy 定义用户、检测记录、报告、规则、黑白名单、插件实例、插件事件、反馈和 AI 配置等模型，通过 Alembic 管理数据库结构演进。

## 风险检测流程

风险检测由 `ScanService` 和 `Detector` 组织，主要流程包括：

1. 规范化 URL 和域名。
2. 检查用户个人策略和全局黑白名单。
3. 提取 URL 与页面行为特征。
4. 执行规则引擎，得到行为风险分数和命中规则。
5. 在触发条件满足时调用 DeepSeek 语义分析。
6. 融合规则分数和 AI 分数，得到最终风险等级。
7. 生成检测记录和报告。

## 规则引擎

规则引擎基于页面特征执行可解释的风险判断。当前规则关注：

- URL 长度和复杂度
- 直接 IP 访问
- 可疑子域名
- 高风险路径词
- 密码输入框
- 跨域表单提交
- 风险诱导关键词
- 品牌仿冒
- 标题与域名不匹配
- 可疑跳转提示
- 凭证窃取组合特征
- 支付、钱包和验证码诱导特征

规则命中后会返回规则名称、风险分值、证据和解释，便于报告展示和管理员复核。

## DeepSeek 语义分析

DeepSeek 用于补充规则引擎无法完全覆盖的语义风险判断。系统不会将完整网页源码、Cookie、localStorage 或表单内容发送给 AI 服务，而是发送经过结构化和截断处理的页面特征。URL 中的敏感 token、邮箱、手机号、证件号、银行卡样式文本和长随机字符串会被脱敏。

当 DeepSeek 成功返回分析结果时，系统综合行为规则分数和语义风险分数计算最终分数；当 DeepSeek 未配置、未触发、超时或异常时，系统自动退回规则引擎结果，保证检测流程可用。

## 前端实现

前端使用 React + TypeScript + Vite 实现，页面由 React Router 管理。前端服务层通过 Axios 调用后端接口，并统一处理登录状态、访问令牌刷新和接口错误。

前端采用组件化结构，常用组件包括表格、状态提示、风险标签、统计卡片、页面标题、加载块和规则命中列表。

## 插件实现

插件基于 Manifest V3 实现，后台 Service Worker 监听标签页变化并触发检测。content script 负责从网页中提取轻量页面特征。插件本地使用 `chrome.storage.local` 保存设置、策略快照、当前标签页状态和临时信任记录。

插件与后端之间使用插件实例 ID、插件版本和插件专用令牌进行绑定和识别。管理员可以撤销插件实例，撤销后对应插件需要重新绑定。

## 数据安全设计

- Web Refresh Token 使用 HttpOnly Cookie 保存，服务端只保存哈希。
- 插件 Refresh Token 与插件实例绑定，撤销实例后令牌失效。
- 生产模式要求强 JWT 密钥、安全 Cookie、精确 CORS 白名单和关闭开发鉴权。
- AI 请求只发送必要结构化特征，避免上传完整网页和浏览器敏感数据。
