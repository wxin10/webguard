# 07-测试说明

## 测试目标

测试目标是验证 WebGuard 的后端接口、前端构建、浏览器插件构建、核心检测流程和工程集成能力。

## 后端测试

后端测试使用 pytest，覆盖鉴权、检测、规则引擎、插件绑定、AI 配置、反馈、威胁情报解析和主要 API。

运行方式：

```powershell
cd backend
python -m pytest
```

主要测试内容包括：

- 用户登录、注册、刷新令牌和退出登录。
- URL 检测和页面检测接口。
- 规则引擎命中、评分和规则 DSL。
- DeepSeek 分析服务的异常和回退处理。
- 插件绑定挑战、令牌交换、刷新和撤销。
- 插件策略同步和事件上报。
- 管理员规则、域名、用户和 AI 配置接口。
- 外部威胁情报源解析和同步逻辑。

## 前端测试

前端主要通过静态检查和构建验证进行质量控制。

```powershell
cd frontend
npm run lint
npm run build
```

验证内容包括：

- TypeScript 类型检查。
- React Hook 使用规范。
- ESLint 规则检查。
- Vite 生产构建。
- 页面模块和 service 调用是否可正确编译。

## 插件测试

插件通过 TypeScript 编译和构建验证：

```powershell
cd extension
npm run build
```

验证内容包括：

- Manifest V3 入口脚本可编译。
- background、content、popup、options、warning 页面脚本可编译。
- HTML 和 CSS 可复制到 `dist/`。
- 插件目录可被浏览器以已解压扩展形式加载。

## 手动联调流程

1. 启动 PostgreSQL、后端和前端。
2. 打开 `http://127.0.0.1:5173/login`。
3. 使用已创建账号登录。
4. 打开 URL 检测页面，输入 `https://example.com`，确认返回安全或低风险结果。
5. 输入带有高风险特征的 URL，例如包含 `login`、`verify`、`password`、仿冒品牌词的地址，确认返回可疑或恶意结果。
6. 打开检测记录页面，确认记录已保存。
7. 打开报告详情页，确认规则命中、风险分数和建议可展示。
8. 加载浏览器插件，配置后端和 Web 地址。
9. 使用插件扫描网页，确认插件状态、风险提示页和报告入口可用。
10. 在风险提示页选择临时信任或永久信任，确认 Web 平台策略中可查看相关记录。

## 持续集成

仓库中的 GitHub Actions 会在代码推送和 Pull Request 时执行：

- 后端 pytest。
- 前端 lint 和 build。
- 插件 build。

CI 使用 SQLite 作为轻量测试数据库，正式运行建议使用 PostgreSQL。
