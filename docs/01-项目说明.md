# 01-项目说明

## 项目名称

WebGuard 恶意网站检测与安全预警系统。

## 项目定位

WebGuard 面向网页访问场景，提供恶意网站检测、风险预警、检测记录留存、黑白名单管理和安全策略联动能力。系统采用“网站平台为主体、浏览器插件为辅助”的产品形态：Web 平台负责用户操作、管理员管理、报告展示和策略配置；浏览器插件负责在用户访问网页时进行轻量采集、风险提醒和访问侧处置；后端服务负责风险检测、AI 分析、鉴权、持久化和策略判定。

## 建设背景

钓鱼网站、仿冒登录页面、虚假支付页面、恶意跳转页面和诱导输入敏感信息的网站在日常网络访问中较为常见。传统黑名单方式难以覆盖不断变化的新网站，仅靠浏览器本地判断也难以形成完整的记录、复盘和管理闭环。WebGuard 通过规则检测与 DeepSeek 语义分析结合的方式，对网页风险进行多维判断，并通过 Web 平台和浏览器插件形成联动防护。

## 建设目标

- 建立可运行的网页风险检测系统，支持 URL 检测和页面特征检测。
- 建立 Web 平台，支持用户查询风险、查看记录、查看报告和维护个人策略。
- 建立浏览器插件，支持访问网页时自动检测、风险提醒、阻断提示和策略同步。
- 建立管理员后台，支持规则、域名、样本反馈、插件实例、用户和 AI 配置管理。
- 建立检测记录和报告机制，便于追踪风险来源、命中规则和处置动作。
- 建立可扩展的后端架构，为后续接入更多威胁情报、模型能力和运维能力留出空间。

## 适用场景

- 校园网络安全课程实践和竞赛作品展示。
- 钓鱼网站识别、恶意 URL 检测和网页安全实验。
- 中小型组织的网页访问安全辅助防护。
- 浏览器插件与 Web 平台联动的安全产品原型。

## 系统角色

- 普通用户：使用 Web 平台进行风险查询，查看个人检测记录和报告，维护个人信任或阻止域名。
- 管理员：维护检测规则、黑白名单、样本反馈、插件配置、AI 配置和用户信息。
- 浏览器插件：作为用户浏览器侧辅助客户端，负责实时检测、策略缓存、风险提醒和事件上报。
- 后端服务：作为可信判定中心，负责检测、策略、报告、鉴权和数据持久化。

## 成果形态

WebGuard 由三部分组成：

- Web 管理平台：基于 React 和 TypeScript 实现。
- 后端检测服务：基于 FastAPI、SQLAlchemy、Alembic 和 PostgreSQL 实现。
- 浏览器插件：基于 Manifest V3 和 TypeScript 实现。
