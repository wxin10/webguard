# 06-安装与运行说明

## 环境要求

- Python 3.11 或更高版本
- Node.js 18 或更高版本
- PostgreSQL 15 或兼容版本
- Chrome 或 Edge 浏览器

## 数据库准备

创建 PostgreSQL 用户和数据库：

```sql
CREATE USER webguard WITH PASSWORD 'webguard';
CREATE DATABASE webguard OWNER webguard;
```

后端通过 `DATABASE_URL` 连接数据库：

```powershell
$env:DATABASE_URL = "postgresql://webguard:webguard@127.0.0.1:5432/webguard"
```

## 后端安装与启动

```powershell
cd backend
python -m pip install -r requirements.txt
$env:DATABASE_URL = "postgresql://webguard:webguard@127.0.0.1:5432/webguard"
alembic upgrade head
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

健康检查地址：

```text
http://127.0.0.1:8000/health
```

创建管理员账号：

```powershell
cd backend
$env:WEBGUARD_SEED_USERNAME = "admin"
$env:WEBGUARD_SEED_PASSWORD = "admin"
$env:WEBGUARD_SEED_ROLE = "admin"
$env:WEBGUARD_SEED_EMAIL = "admin@example.local"
$env:WEBGUARD_SEED_DISPLAY_NAME = "系统管理员"
python -m app.scripts.seed_dev_user
```

## 前端安装与启动

```powershell
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

访问地址：

```text
http://127.0.0.1:5173
```

## 浏览器插件安装

```powershell
cd extension
npm install
npm run build
```

加载方式：

1. 打开 Chrome 或 Edge 扩展管理页面。
2. 启用开发者模式。
3. 选择“加载已解压的扩展程序”。
4. 选择项目中的 `extension/` 目录。
5. 在插件 Options 页面填写后端地址和 Web 平台地址。

本地默认地址：

```text
API Base URL: http://127.0.0.1:8000
Web App URL:  http://127.0.0.1:5173
```

## 局域网 IP 访问

后端监听所有网卡：

```powershell
cd backend
$env:DATABASE_URL = "postgresql://webguard:webguard@127.0.0.1:5432/webguard"
$env:CORS_ORIGINS = "http://192.168.1.10:5173,http://127.0.0.1:5173"
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

前端监听局域网：

```powershell
cd frontend
$env:VITE_API_BASE_URL = "http://192.168.1.10:8000"
npm run dev -- --host 0.0.0.0 --port 5173
```

浏览器访问：

```text
http://192.168.1.10:5173
```

其中 `192.168.1.10` 替换为实际服务器或电脑 IP。

## 服务器部署要点

服务器部署时建议：

- 使用 PostgreSQL 作为正式数据库。
- 设置强 `JWT_SECRET`。
- 设置 `DEBUG=false`。
- 设置 `ENABLE_DEV_AUTH=false`。
- 设置 `ENABLE_RUNTIME_SCHEMA_GUARD=false`。
- 设置 `REFRESH_TOKEN_COOKIE_SECURE=true`。
- 配置 HTTPS 和反向代理。
- 配置精确的 `CORS_ORIGINS`。
- 前端构建时设置 `VITE_API_BASE_URL` 为服务器 API 地址。
- 浏览器插件发布前根据服务器地址调整 `host_permissions` 和插件 Options 默认地址。

前端构建：

```powershell
cd frontend
$env:VITE_API_BASE_URL = "https://api.example.com"
npm run build
```

后端启动示例：

```powershell
cd backend
$env:DATABASE_URL = "postgresql://user:password@db-host:5432/webguard"
$env:DEBUG = "false"
$env:ENABLE_DEV_AUTH = "false"
$env:ENABLE_RUNTIME_SCHEMA_GUARD = "false"
$env:JWT_SECRET = "<至少32位的强随机密钥>"
$env:REFRESH_TOKEN_COOKIE_SECURE = "true"
$env:CORS_ORIGINS = "https://web.example.com"
alembic upgrade head
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```
