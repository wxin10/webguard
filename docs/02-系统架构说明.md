# 02-系统架构说明

## 总体架构

WebGuard 采用前后端分离和浏览器插件联动架构，整体分为 Web 平台、后端服务、浏览器插件和数据库四层。

```text
用户浏览器
├─ Web 平台：登录、查询、报告、策略、管理员后台
└─ 浏览器插件：实时检测、风险提醒、访问侧操作

后端服务
├─ 鉴权与用户服务
├─ 风险检测服务
├─ 规则引擎
├─ DeepSeek 语义分析服务
├─ 报告与记录服务
├─ 黑白名单与策略服务
└─ 插件绑定与事件服务

数据层
└─ PostgreSQL：用户、规则、检测记录、报告、策略、插件实例、AI 配置
```

## 架构原则

- 后端是风险判定和策略管理的可信中心。
- Web 平台承担主要管理能力，插件保持轻量。
- 插件可以进行本地缓存和快速预处理，但最终风险判断以后端为准。
- 所有核心数据进入数据库，便于查询、审计、复盘和统计。
- API 响应格式统一，降低 Web、插件和后端之间的联调成本。

## Web 平台架构

前端使用 React、TypeScript 和 Vite 构建，采用 React Router 组织页面路由，使用 Axios 访问后端接口。主要页面包括：

- 产品首页
- 登录与注册
- 用户工作台
- URL 检测页
- 我的检测记录
- 最新报告和报告详情
- 个人域名策略
- 插件绑定和插件同步状态
- 管理员记录、样本、规则、域名、AI 配置、统计、插件和用户管理页面

## 后端服务架构

后端使用 FastAPI 构建 REST API，使用 SQLAlchemy 管理 ORM 模型，使用 Alembic 管理数据库迁移。后端模块主要包括：

- `api/`：定义接口路由。
- `schemas/`：定义请求和响应数据结构。
- `models/`：定义数据库模型。
- `services/`：实现检测、策略、报告、插件、用户、AI 配置等业务逻辑。
- `core/`：配置、数据库连接、统一响应、异常处理、鉴权上下文和安全工具。

## 浏览器插件架构

浏览器插件基于 Manifest V3 实现，主要模块包括：

- `background.ts`：负责监听标签页更新、触发自动检测、处理消息和跳转风险提示页。
- `content.ts`：负责采集当前网页标题、可见文本、按钮文本、输入框提示、表单提交域名和密码输入框等特征。
- `popup/`：展示当前页面安全状态，提供手动扫描和快速操作入口。
- `options/`：配置后端地址、Web 平台地址、插件绑定和检测开关。
- `warning/`：展示风险提示、查看报告、返回安全页面、临时信任或永久信任当前网站。
- `utils/`：封装 API、存储、检测结果解析和导航工具。

## 数据流

典型实时检测流程如下：

1. 用户在浏览器中访问网页。
2. 插件监听页面加载完成，判断是否需要自动检测。
3. 插件 content script 采集页面特征。
4. 插件将页面特征发送到后端检测接口。
5. 后端先检查用户策略、全局黑白名单和威胁情报。
6. 未命中确定性策略时，后端提取行为特征并执行规则引擎。
7. 高风险条件触发时，后端调用 DeepSeek 进行语义分析。
8. 后端融合规则分数和 AI 分析结果，生成风险等级和处置建议。
9. 检测记录和报告写入数据库。
10. 插件根据结果展示安全、可疑或恶意页面提示。

## 部署形态

系统支持三种运行形态：

- 本机运行：前端、后端和数据库部署在同一台开发电脑。
- 局域网访问：后端和前端监听局域网地址，其他设备通过 IP 访问。
- 服务器部署：后端连接服务器 PostgreSQL，前端通过 Web 服务发布，插件配置服务器 API 地址。
