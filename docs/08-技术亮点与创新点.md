# 08-技术亮点与创新点

## Web 平台与浏览器插件联动

WebGuard 不是单独的网页扫描工具，也不是单独的浏览器插件，而是将 Web 平台、后端检测服务和浏览器插件组合为完整闭环。Web 平台负责管理和复盘，插件负责访问侧实时提醒，后端负责可信判定和数据持久化。

## 规则引擎与 DeepSeek 语义分析结合

系统先使用规则引擎对 URL 和页面特征进行快速、可解释判断，再在高风险条件触发时调用 DeepSeek 进行语义分析。这样既保留了规则检测的速度和可解释性，也提升了对诱导话术、品牌仿冒、凭证窃取和虚假支付页面的识别能力。

## 可解释风险报告

检测结果不仅给出风险等级，还展示风险分数、命中规则、行为信号、AI 分析状态、页面特征摘要和处置建议。用户和管理员可以理解系统为什么判定页面存在风险。

## 黑白名单与个人策略联动

系统同时支持全局策略和个人策略。管理员维护全局信任与阻止域名，用户可维护个人信任、阻止和临时信任策略。插件同步策略后，可以在访问侧快速进行本地预判断。

## 插件实例绑定机制

浏览器插件通过绑定挑战和插件专用令牌与用户账号关联。插件请求携带插件实例 ID 和插件版本，后端可以识别实例并支持撤销，增强了插件侧访问控制能力。

## AI 配置可管理

DeepSeek 接入不是写死在代码中，而是支持管理员在 Web 后台配置和测试。系统同时保留环境变量作为兜底配置，便于不同部署环境调整。

## 敏感信息保护

AI 分析只接收结构化页面特征，不上传完整 HTML、Cookie、localStorage 或表单内容。系统会对 URL 中的敏感 token、邮箱、手机号、证件号和长随机字符串进行脱敏，减少隐私风险。

## 面向扩展的工程结构

后端采用 api、schemas、models、services、core 分层；前端采用页面、布局、组件、service 和 context 分层；插件按 background、content、popup、options、warning 和 utils 分层。结构清晰，便于继续扩展威胁情报、规则体系、模型能力和运维能力。

## 适合竞赛展示的完整闭环

系统可以展示从“用户访问网页”到“插件实时检测”、再到“后端生成报告”、最后到“Web 平台复盘和管理员审核”的完整流程，符合计算机设计大赛对作品完整性、工程实现和应用价值的要求。
